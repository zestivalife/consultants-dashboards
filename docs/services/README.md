# Services Documentation Index

Each deployable service should eventually expose the following documentation set:

- `README.md`
- `SERVICE_SPECIFICATION.md`
- `API.md`
- `DATA_OWNERSHIP.md`
- `RUNBOOK.md`
- `ENVIRONMENT.md`
- `RELEASE_NOTES.md`
- `INCIDENTS.md`

Do not invent service behavior in placeholders.
